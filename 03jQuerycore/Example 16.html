Chaining

If you call a method on a selection and that method returns a jQuery object,
you can continue to call jQuery methods on the object without pausing for
a semicolon. This practice is referred to as "chaining":
example:

$("#content").find("h3").eq(2).html("new text for third h3");

this happens to the following div with id = content.

the first h3

The second h3

third h3

the 4th h3



It may help code readability to break the chain over several lines:

$("#content")
.find("h3")
.eq(2)
.html("new text for third h3");


jQuery also provides the .end() method to get back to the original
selection should you change the selection in the middle of a chain:

example on a div with id = content2:

$("#content2")
.find("h3")
.eq(2)
.html("new text for third h3")
.end()
.eq(0)
.html("new text for first h3");
